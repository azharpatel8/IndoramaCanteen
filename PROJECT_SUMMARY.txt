================================================================================
API ENDPOINTS REFERENCE
================================================================================

Auth:
  POST   /api/auth/register         - Register new user
  POST   /api/auth/login            - Login user

Menu:
  GET    /api/menu                  - Get all menu items
  GET    /api/menu/:id              - Get item by ID
  GET    /api/menu/category/:cat    - Get items by category

Orders (Protected):
  POST   /api/orders                - Create order
  GET    /api/orders                - Get user orders
  GET    /api/orders/:id            - Get order details
  PUT    /api/orders/:id/cancel     - Cancel order

Billing (Protected):
  POST   /api/billing               - Process payment
  GET    /api/billing               - Get bills
  GET    /api/billing/:id           - Get bill details

Feedback (Protected):
  POST   /api/feedback              - Submit feedback
  GET    /api/feedback              - Get feedback
  GET    /api/feedback/category/:cat- Get feedback by category

================================================================================
DATABASE SCHEMA
================================================================================

users (7,000+ records capacity)
├── user_id (PK, auto-increment)
├── username (unique)
├── email (unique)
├── password_hash (bcrypt)
├── full_name
├── role (employee)
├── department
└── timestamps (created_at, updated_at)

menu_items (unlimited items)
├── item_id (PK, auto-increment)
├── name
├── description
├── category (Main Course, Beverages, Desserts, Snacks)
├── price
├── available_quantity
├── is_available (boolean)
└── image_url

orders (unlimited capacity)
├── order_id (PK, auto-increment)
├── user_id (FK)
├── order_date
├── status (pending, confirmed, cancelled)
├── total_amount
├── special_instructions

order_items (tracks line items)
├── order_item_id (PK)
├── order_id (FK)
├── item_id (FK)
├── quantity
└── unit_price, subtotal

billing (payment records)
├── bill_id (PK, auto-increment)
├── order_id (FK)
├── user_id (FK)
├── amount
├── payment_method (cash, card, digital)
└── payment_status (completed, pending)

feedback (reviews & ratings)
├── feedback_id (PK, auto-increment)
├── user_id (FK)
├── order_id (FK, optional)
├── rating (1-5)
├── category (service, food_quality, cleanliness, price)
└── comment (text)
